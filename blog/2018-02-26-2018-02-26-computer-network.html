计算机网络

发表于 2018-02-26 | 分类于 notes

1 计算机网络概论

1.1 计算机网络

1.1.1 定义

由通信信道连接的 主机和网络设备 的集合，以方便用户共享资源和互相通信。

1.1.2 特征

  • 最主要特征 是通用性，没有对特定应用进行优化
  • 能够支持广泛的、不断出现的新型应用

1.1.3 用途

  • 共享信息
  • 协同计算
  • 方便通信
  • 共享硬件

1.1.4 组成

  1. 称为节点的计算设备
    1. 端系统（主机）
    2. 中间节点

      路由器，交换机，自治系统，虚拟节点等。

  2. 称为链路的物理媒体

1.1.5 连接方式

  • 直接连接
  • 网络云
  • 网络云互联

1.1.6 传递数据的方式

  1. 电路交换（电话网）

    基本过程

    1. 建立连接
    2. 传输数据
    3. 连接释放

    优点 独占资源，通信质量有保证，延迟小，无拥塞 缺点 建立连接时间长，工作复杂，有时工作效率低

  2. 分组交换（计算机网络）
    • 首先采用存储转发机制
    • 将长数据划分为若干短数据形成分组

    以上两条为 计算机网络最根本工作机制

  3. 报文交换

    优点 无连接，采用存储转发 缺点 对报文大小无限制，有时占用中间节点空间大，延时大

1.1.7 网络拓扑结构

分类 优点 缺点
总线型 结构简单 单点故障可能影响全网
星型 扩展方便 对中心节点依赖较大
树型 扩展方便 对非叶子节点依赖较大
环形 结构简单，扩展方便 单点故障可能影响全网
网状 扩展方便 冗余太多

1.1.8 网络服务

向用户提供有用的网络功能，由联网的端系统中的网络应用程序通过相互协作来提供。

1.2 因特网

1.2.1 组成

  1. 因特网边缘
  2. 接入网
  3. 因特网核心

1.2.2 特点

边缘复杂，核心简单 ：遵循“端到端”的原则，即将复杂的网络处理功能置于因特网边缘，而将相对简单的分组交付功能放在因特网核心。 优点 ：充分利用了位于因特网边缘的端系统强大的计算能力，用软件方式处理大量复杂的控制和应用逻辑，而使位于因特网核心的路由器尽可能简单，以高速地转发分组。

1.2.3 成功的因素

在网络边缘用软件实现，方便增加新的网络功能

1.2.4 应用程序体系结构

  1. 客户/服务器体系结构 （C/S模式）
    1. 服务器
      1. 运行服务器程序的主机永远在运行
      2. 具有永久IP和固定端口号
      3. 响应客户机程序的请求，并提供服务
    2. 客户机
      1. 向服务器发出请求，接收服务器响应
      2. 可以间歇与服务器连接
      3. 具有动态IP地址和随机端口号
      4. 客户机之间不通信
  2. P2P体系结构（P2P模式）
    1. 特点
      1. 能够借助对等方的空闲资源而不需要庞大的服务器基础设施的支持，从而降低服务提供商成本
      2. P2P应用具有群组性、高度分布性和动态性
      3. P2P体系结构具有自扩展性

1.2.5 接入网

指将端系统连接到其边缘路由器的物理链路及设备的集

1.2.6 分类

  1. 从作用范围
    1. 广域网 WAN
    2. 局域网 LAN
    3. 城域网 MAN
    4. 接入网 AN
  2. 从使用者
    1. 专用网
    2. 公用网

1.3 协议分层与服务模型

1.3.1 协议

为进行网络中的数据交换而建立的规则、标准或规约称为网络协议

  1. 三要素
    1. 语法：数据与控制信息的结构与格式
    2. 语义：语法定义结构中的具体含义
    3. 语时：事件实现顺序的详细说明

1.3.2 分层的网络体系结构

计算机网络的各层及相应协议的集合称为网络的体系结构

  1. 分层的优点
    1. 简化系统设计
    2. 模块化易于维护系统的更新
  2. 分层的缺点

    功能冗余，降低系统性能；而且分层越多，分组首部越大

1.3.3 报文的层间传递

同一系统相邻两层的实体进行交互的地方称为服务访问点 SAP

  1. 水平协议和垂直服务
    1. 向上层提供服务
    2. 协议是“水平的”，服务是“垂直的”
    3. 向上的服务利用了下层的功能
  2. 多层通信的实质
    1. 对等层实体之间虚拟通信
    2. 下层向上层提供服务
    3. 上层使用下层提供的服务
    4. 实际通信在最底层媒体完成

1.3.4 因特网体系结构

协议数据单元PDU是OSI参考模型中对等层协议之间交换的信息单元

  1. OSI（开放系统互联）参考模型
    自顶向下 信息单位
    应用层 无
    表示层 无
    会话层 无
    传输层 数据段、报文
    网络层 分组、数据报
    数据链路层 数据帧
    物理层 比特
  2. TCP/IP体系结构
    1. 特点
      1. 协议栈呈“沙漏型”，IP是体系结构的“细腰”，代表最小的、精选的通用功能集，能够使高层应用与底层通信网技术独立发展
      2. 为了提高系统效率没有严格分层
    2. 重点
      1. 边缘节点必须实现TCP/IP所有五个层
      2. 交换机只有两层（物理层和数据链路层）
      3. 路由器只有三层（物理层，数据链路层和网络层）
    3. 两个子网
      1. 资源子网：应用层和运输层
      2. 通信子网： 物理层，数据链路层和网络层

1.4 分组交换网的性能指标

1.4.1 重要概念

  1. 带宽
    1. 在通信领域指信号具有的频带宽度，单位是赫兹
    2. 指链路在一定时段内所能传输的比特数的额定值，常用单位时Kb/s等（十的次方）
  2. 时延
    1. 处理时延
    2. 排队时延
    3. 传输时延（等于分组长度除以链路带宽）
    4. 传播时延（等于节点间距离除以传播速度）

2 数据通信基础

2.1 数据通信基础知识

2.1.1 通信系统模型

通信的目的 是用电信手段传递报文中的信息。 通信系统 的作用就是将信息从 信源 发送到一个或多个 信宿 。

  1. 组成

    信源，发送设备，信道，接收设备，信宿

2.1.2 数字通信及其性质

数据 是运输信息的实体， 信号 是数据的电气或电磁表现。 数据是针对信息而言的，而信号则是针对传输而言的

  1. 数字信号和模拟信号

    前者是离散的，后者是连续的。将数字信号转化为模拟信号的过程称为 调制 ，反之称为 解调制 。

    1. 数字信号

      基本单位是 码元 。数字通信具有抗干扰能力强且噪声不积累，传输差错可控，便于对数字信息进行处理、变换、存储，易于复用和易加密处理等优点。

    2. 奈奎斯特公式

      C=2W log2 M bps （低通带“2”，带通不带“2”,W为信道带宽，M为信号状态数量）

    3. 香农公式

      C=W log2 (1+S/N) bps （W为信道带宽，S为信道内所传信号的平均功率，N为信道内部的高斯噪声功率）

2.1.3 通信方式

  1. 三种基本方式
    1. 单工通信：只能单方向传输报文
    2. 半双工通信：能够收发报文，但不能同时进行
    3. 全双工通信：能够同时收发报文
  2. 两种信号

    基带 信号和 宽带 信号

2.2 传输媒体

2.2.1 物理层基本概念

主要任务为确定与传输媒体的接口的一些特性

  1. 机械特性

    指明接口所用接线器的形状和尺寸、引线数目和排列、固定和锁定装置等

  2. 电气特性

    指明在接口电缆的各条线上出现的电压的范围

  3. 功能特性

    指明某条线上出现的某一种电平的电压表示何种意义

  4. 规程特性

    指明对于不同功能的各种可能事件的出现顺序

2.2.2 导向传输媒体

  1. 双绞线
  2. 光纤

    由光缆、光源和检测器组成

  3. 同轴电缆
    1. 粗缆：10BASE5（表示10M基带，最长500米）
    2. 细缆：10BASE2
    3. 100BASEF，100BASEB（F表示光纤，B表示宽带）

2.2.3 非导向传输媒体

微波，卫星和无线电

2.3 编码和复用

2.3.1 编码

为了使信道能有效地传输信号，并且接收节点能够有效地识别发送节点发送的比特， 首先要将比特编码为信号 ，然后再在媒体上进行传输。

  1. 不归零（NRZ）

    将比特数值1映射为高电平信号，数值0映射为低电平信号。但连续的1或0会带来两个问题。

    1. 基线漂移：连续的1或0影响信号平均值，从而难以检测出信号变化
    2. 时钟恢复：发送方和接收方不能同步时钟，无法正确对信号进行解码
  2. 不归零反转（NRZI）

    将当前信号的一个跳变编码为一个1,将保持当前信号编码为0（只解决了连续1的问题，没有解决连续0的问题）

  3. 曼彻斯特编码

    通过传输NRZ编码数据与时钟的异或合并了时钟和信号

2.3.2 多路复用

指在一条物理链路上同时传输多路信息，以扩大信道的传输容量的技术。

  1. 频分复用（FDM）
  2. 时分复用（TDM）
  3. 波分复用（WDM）
  4. 码分复用（CDM）又称码分多址（CMDA）

CMDA中将每一个比特时间再划分为m比特的序列，称为 码片 。CMDA给每个节点分配的码片序列各不相同，并且相互正交。

  1. 任意两个不同节点的码片向量规格化内积为0
  2. 任意一个码片向量和自己的规格化内积为1
  3. 任意一个码片向量和自己反码向量规格化内积为-1

    该协议定义了一种用于解决在分组无线通信系统中分配广播信道的分布式方法。 操作如下：

    1. 当节点获得新帧，将随时发送
    2. 若无碰撞，节点能连续发送新帧
    3. 若碰撞，节点可以检测到
  2. 时隙ALOHA协议

    优点

    1. 单个活跃节点能够连续以信道全速传输
    2. 减少碰撞的概率
    3. 简单

3.6.1 争用期

以太网的端到端往返时延称为争用期或碰撞窗口（用来检测发送的数据帧是否遭受了碰撞）

3.6.2 MAC地址

以太网工作在数据链路层，乙太网适配器具有唯一的链路层地址：局域网地址， 物理地址，硬件地址或MAC地址（在网卡上）

3.6.3 以太网的多路访问协议 CSMA/DA

工作机制 ：

  1. 可以在任意时刻发送帧的非同步机制
  2. 发送前若侦听到其他适配器在正在传输，决不会传输帧的“发前先听”机制
  3. 传输中适配器一旦检测到另一个适配器正在传输，就中止自己传输的“碰撞检测”机制

3.6.4 以太网提供的是不可靠无连接服务

3.7 局域网

用多个中继器或集线器可连接成更大局域网。物理层集线器的拓扑结构，在物理上是星型结构， 在逻辑上是总线型结构。

3.8 网桥

物理层设备扩展网络，冲突域变大

在数据链路层扩展局域网使用网桥（可以将不同的局域网连接起来），网桥工作在数据链路层， 根据MAC帧的目的地址对收到的帧进行转发，因而它具有 转发和过滤帧 的功能，但 网桥不 改变转发的帧的源地址

3.8.1 优点

  1. 过滤通信量
  2. 扩大通信范围
  3. 提高可靠性
  4. 可连接不同物理层、不同MAC层、不同速率的局域网

网桥连接局域网，冲突域不变，但使用集线器连接会扩大冲突域

3.8.2 以太网交换机

  • 采用全双工方式
  • 每个用户在通信时是独占而不是和其他网络用户共享传输媒体的带宽

3.8.3 透明网桥使用支撑树算法

3.9 虚拟局域网

一次路由，多次交换

3.10 IEEE 802.11 无线局域网

主要有两种形式：具有基础设施无线局域网 LAN；无基础设施无线局域网 自组织网络，

3.10.1 无线局域网不采用CSMA/CD协议的原因

  1. CSMA/CD协议要求一个站点在发送本站数据的同时还必须不间断地检测信道，但在无线局域网 的设备中要实现这种功能就花费过大
  2. 即使实现碰撞检测的功能，依然不能避免碰撞

3.10.2 两个重要问题

  1. 隐蔽站问题
  2. 暴露站问题

3.10.3 帧间间隔 IFS

为了避免碰撞，所有的站在完成发送后，必须再等待一段很短的时间（继续监听）才能发送下一 帧。这段时间称为IFS

3.10.4 避免碰撞机制

采用CMSA/CA协议，并用RTS和CTS帧进行信道预约， 用短预约分组尝试预约信道，避免长数据 帧的碰撞，提高了协议效率

4 网络互联

4.1 网络层概述

网络层利用直接连接的网络向上层所提供的链路层服务，基于本层协议，提供了 将分组从网络 一侧的主机传输到网络另一侧的主机的端到端通信服务

4.1.1 异构网络互联方法

在体系结构和通信协议方面具有很大的差异的计算机网络，称为 异构网络

异构网络的互联方法

  1. 两两互联
  2. 与中立的中继系统互联

因特网采用了一种与机器无关、操作系统无关、语言无关的中继系统（IP系统）方式来解决异 构网络互联问题

  1. IP网络具有统一的协议和地址，IP地址标识了主机在网络中的位置，为分组全局性寻址和路 由选择，进而实现网络互联互通提供了可能
  2. IP网络是虚拟网络，为分组提供全局性路由选择和转发功能，分组的实际传输交付功能却要 由下面的通信网络提供
  3. 连接各个异构网络的设备是路由器，最高层次是网络层，至少有两个与不同网络连接的接口。 全局性路由选择和转发分组功能由路由器实际完成
  4. 端系统位于网络路径两侧的端点，具有协议栈的所有层次。在端系统和沿路径各路由器网络 层的共同作用下，才形成了主机到主机的端到端路径

4.1.2 关键的网络层功能

转发 将分组从路由器的输入移动到适当的路由器输出

选路 决定分组从源到目的地所采用的路由 路由选择算法

路由 分组从源到目的地所经过的端到端路径

4.1.3 分组直接交付和间接交付

当源主机和目的主机位于同一个直接连接的网络上时，分组从一台源主机直接传输到另一台目的 主机的过程称为 直接交付

当目的主机不与源主机在一个直接连接的网络上时，源主机必须先把分组发给一台路由器，由路 由器进行后继处理，然后才能把分组交付给目的主机，这个过程称为 间接交付

分组转发算法

4.2 网络服务模型

仅在网络层提供连接服务的计算机网络被称为 虚电路网络 ；仅在网络层提供无连接服务的计 算机网络被称为 数据报网络

4.2.1 数据报格式

最大传送单元 一个链路层帧能承受的最大数据量为MTU，当数据报长度超过MTU时需要进行 IP分片

4.2.2 IP编址

IP地址长度为32比特，采用 点分十进制记法

  1. 分类IP地址（表示为 网络号＋主机号）
  2. 子网划分
  3. 构成超网（无分类编址方法）

4.2.3 划分子网

IP地址表示为 网络号＋子网号＋主机号，然后用子网掩码找到子网

4.2.4 CIDR划分网络

IP地址表示为 网络地址/前缀，采用 最长前缀匹配规则 解决某个目的地址与转发表中多个项 相匹配的问题。使用单个网络前缀通告多个网络的能力称为 地址聚合 也称为 路由聚合 （构成超网）

4.2.5 IP地址的获取与配置

动态主机配置协议 DHCP

路由器连接多个网络后还是多个网络，而交换机可以将多个网络连接成一个网络。路由器会阻断 广播，而集线器、交换机不能。因此路由器可以用来划分子网

4.2.6 NAT 网络地址转换

3部分专用IP地址空间：10.0.0.0/24,172.16.0.0/24,192.168.0.0/24

使用 NAT转换表 完成地址转换

4.2.7 ARP 地址映射协议

是一种能够从IP地址获取对应MAC地址的协议。两个端系统在同一个子网下直接运行ARP协议获取 对方MAC地址，若不在同一个子网下，需要借助中间路由器多次运行ARP协议

4.3 路由选择协议及其算法

自治系统 处于一个管理机构控制之下的网络和路由器群组。在一个自治系统内部使用的路由 选择协议称为 内部网关协议 IGP 。跨自治系统需要使用 外部网关协议 EGP 。

因特网采用 分层次 的路由选择协议。

4.3.1 RIP 路由选择信息协议

IP网络中使用得最早、应用最广泛的AS内部路由选择协议。 是不分层次的路由选择算法 ，使 用 运输层 的 用户数据报UDP ，端口是520.

三个要点

  1. 仅和相邻路由器交换信息
  2. 交换的信息是当前本路由器所知道的全部信息，即自己的路由表
  3. 按固定的时间间隔交换路由信息

4.3.2 OSPF 开发最短路优先

是分层的路由选择算法

三个要点

  1. 向本自治系统中所有路由器发送消息，这里使用的是洪泛法
  2. 发送的是与本路由器相邻的所有路由器的链路状态，但这只是路由器所知道的部分信息
  3. 只有当链路状态发生变化时，路由器才用洪泛法向所有路由器发送此信息

Dijkstra算法
