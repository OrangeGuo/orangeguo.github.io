2 hibernate框架

  1. 对jdbc进行封装
  2. 开源轻量级框架
  3. 应用在javaEE的dao层

2.1 ORM思想

即对象关系映射

  1. 让实体类（javabean的别名）和数据库进行一一对应：实体类和数据库表对应；实体类属性 和表里字段对应
  2. 不需要直接操作数据库表，而操作表对应实体类对象

2.2 搭建hibernate环境

  1. 导入hibernate框架jar包，以及日志输出和数据库驱动jar包
  2. 创建实体类（实体类必须包含一个唯一属性）
  3. 配置实体类和数据库表一一对应关系（映射）
  4. 创建hibernate核心配置文件

3.2 一站式框架

spring针对javaee三层结构分别提供不同解决方案

  • web层 springMVC
  • service层 spring的IOC
  • dao层 spring的jdbcTemplate

3.3 bean 实例化三种方式实现

  1. 使用类的无参构造创建
  2. 使用静态工厂创建（创建静态方法返回类对象）
  3. 使用实例工厂创建（创建非静态方法返回类对象）

3.4 属性输入的三种方式

  1. set方法
  2. 有参数构造方法
  3. 使用接口注入

3.5 IOC和DI

IOC：控制反转，把对象创建交给spring配置 DI：依赖注入，向类里面属性设置值 关系：DI不能单调操作，必须在IOC基础上完成操作

3.6 spring整合web项目原理

  1. 加载spring核心配置文件（要new对象，功能可以实现，但效率低）
  2. 优化思路：把加载配置文件和创建对象过程，放在服务器启动时候完成
  3. 实现原理：ServletContext对象，监听器。在服务器启动时，为每个项目创建一个 ServletContext对象，使用监听器可以监听对象创建时间，然后就加载spring配置文件，创 建配置文件配置对象。把创建的对象放到ServletContext域对象里（setAttribute方法）， 获取对象时候，到ServletContext域中获取（getAtrribute方法）

3.7 AOP

面向切面编程，实现拓展功能不修改源代码。采用 横向抽取机制 ，取代传统纵向继承体系重复性代码。
